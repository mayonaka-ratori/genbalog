# Step 1: エンドツーエンドテスト

デモシナリオ:
作業員役がマイクで話す:「3階の足場で板がズレて、バランス崩しかけた」

期待される流れ:

1. AI:「お疲れ様です。報告を受け付けました。なるほど、確認させてください。高さは何メートルくらいでしたか？」
2. 作業員:「7メートルくらいです」
3. AI:「足場での板ズレですね。4M分類は Man と Method、リスクレベルは HIGH です。過去にも同様の報告があります。現場監督に共有されました。報告ありがとうございます。安全第一で作業を続けてください」
4. dashboard/index.html に赤いカードが出現

## 追加テスト: barge-in 対応

AIが分類結果を読み上げている途中で、マイクに向かって
「あ、あともう一つ、釘も落ちてきた」と割り込む。

期待される動作:

1. AIが即座に発話を止める
2. AIが「釘の飛来ですね、それも報告に追記しました」のように応答する
3. Supabase の reports に追記情報が反映されている

これも3回連続成功で合格。

# Step 3: フォールバック対応（各機能が動かない場合）

- Milvus 不調 → milvus_client.py の search_similar を seed_cases.json のテキスト部分一致に差し替え。pymilvus の import 削除。
- function calling 不調 → LLM 出力テキストから正規表現で category_4m と risk_level を抽出し supabase_client.py で直接 INSERT
- TTS が遅い/無音 → TTS を無効化しテキストチャットのみで進行
- ダッシュボード Realtime 不発 → 5秒ポーリング（setInterval で GET）に切り替え
